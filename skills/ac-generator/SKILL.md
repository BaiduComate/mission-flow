# AC Generator — 测试验证点生成器

基于需求上下文，生成高信息密度的功能验证点。

## 工作流

### 第一步：解析输入，拉取需求内容

**前提：当前改动必须涉及需求功能改动，否则跳过验证点生成，不与用户交互。**

根据用户提供的输入类型分别处理：

- **iCafe 卡片编号**（如 `PROJECT-1234`）：拉取 iCafe 卡片详情（标题、需求描述、验收条件）
- **知识库链接**（`ku.baidu-int.com` 链接）：读取知识库文档内容
- **纯文本需求描述**：直接使用，无需拉取

### 第二步：生成测试验证点

**仅基于需求内容**生成，严格遵循 `references/ac-template.md` 中的格式规范与质量要求。

生成前先读取 `references/ac-template.md` 了解：输出格式、数量控制规则、好/坏示例、排除类型。

### 第三步：格式化输出

按 `references/ac-template.md` 的输出格式直接输出：

```markdown
## 功能验证点

1. [触发条件]，[预期结果]
2. [触发条件]，[预期结果]
...
```

输出后告知统计：
> ✅ 共生成 N 个功能验证点

## 关键原则

1. **格式规范**：严格遵循 references/ac-template.md，句式为「[触发条件]，[预期结果]」
2. **仅基于需求**：不依赖技术实现方案，不擅自扩展范围
3. **精简克制**：10 条以内，合并同类，宁少勿多
4. **主动补全**：输入不足时主动拉取需求内容，不向用户当伸手党
