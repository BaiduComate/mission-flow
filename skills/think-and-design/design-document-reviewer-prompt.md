# 设计文档审查提示词模板

派发设计文档审查 subagent 时使用此模板。

**目的：** 验证设计文档是否完整、一致，并已准备好进入实现计划阶段。

**派发时机：** 设计文档已写入 `.comate/specs/{feature_name}/doc.md` 之后。

```
Task tool (general-purpose):
  description: "Review design document"
  prompt: |
    你是设计文档审查员。请验证该设计文档是否完整并已准备好进入计划阶段。

    **要审查的设计文档：** [DESIGN_DOCUMENT_FILE_PATH]

    ## 检查内容

    | 类别 | 关注点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、“TBD”、不完整章节 |
    | 一致性 | 内部矛盾、冲突需求 |
    | 清晰度 | 需求是否含糊到足以导致别人构建错误内容 |
    | 范围 | 是否足够聚焦，能形成单个计划，而不是覆盖多个独立子系统 |
    | YAGNI | 未请求功能、过度工程 |

    ## 校准标准

    **只标记那些会在实现计划阶段造成真实问题的事项。**
    缺失章节、矛盾，或某个需求含糊到可能被解释成两种不同方式，才属于问题。
    轻微措辞改进、风格偏好，以及“某些章节没有其他章节详细”不属于问题。

    除非存在会导致计划缺陷的严重缺口，否则批准通过。

    ## 输出格式

    ## 设计文档审查

    **状态（Status）：** Approved | Issues Found

    **问题（Issues，如有）：**
    - [Section X]: [具体问题] - [为什么会影响计划]

    **建议（Recommendations，仅供参考，不阻塞批准）：**
    - [改进建议]
```
